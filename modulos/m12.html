Conhecer Ana

Ana acompanha este módulo

Ana já aprendeu que a Inteligência Artificial pode gerar textos, responder perguntas, resumir documentos, classificar mensagens e apoiar tarefas do cotidiano. No Módulo 11, ela conheceu os conceitos de agentes de IA, skills, tools, automação inteligente, guardrails, custos e supervisão humana.

Neste módulo, Ana passa a construir automações reais usando o n8n. Ela percebe que escrever um prompt não é o mesmo que criar uma automação. Um prompt é uma instrução enviada para uma IA; uma automação completa pode envolver entrada de dados, validação, transformação, consulta a sistemas, uso de IA, envio de mensagens, geração de documentos, registro em planilhas, logs, tratamento de erros, controle de custos, segurança, LGPD, guardrails e revisão humana.

Ao longo do módulo, Ana aprende que nem tudo deve ser resolvido com IA. Muitas tarefas podem ser resolvidas com regras simples, condições, planilhas, APIs, workflows e pipelines. A IA deve ser usada quando realmente agrega valor, especialmente em situações que envolvem linguagem natural, resumo, classificação, extração de informações ou geração de respostas personalizadas.

1. Apresentação do módulo

O Módulo 12 — Automação Inteligente com n8n, Pipelines, Skills e LLMs é um módulo prático do UNESP.IA, voltado à criação de automações inteligentes utilizando o n8n como ambiente principal de orquestração.

A proposta do módulo é ensinar os participantes a construir workflows que integrem formulários, planilhas, e-mails, documentos, APIs, webhooks, modelos de linguagem, pipelines de dados, skills reutilizáveis, agentes simples, logs, guardrails e revisão humana.

O módulo não se limita ao uso operacional da ferramenta. O objetivo é desenvolver a capacidade de pensar processos, modelar fluxos, organizar dados, decidir quando usar IA, criar skills reutilizáveis, controlar custos, proteger informações e avaliar riscos antes de automatizar ações reais.

12.1 Fundamentos de automação inteligente com n8n

Carga horária

2 horas.

Objetivo da unidade

Compreender o que é automação inteligente, qual é o papel do n8n e por que a construção de automações deve começar pelo entendimento do processo, e não pela escolha da ferramenta.

Situação-problema

A equipe do UNESP.IA recebe inscrições por formulário. Depois disso, precisa registrar participantes, enviar confirmações, organizar listas de presença, responder dúvidas, gerar certificados e criar relatórios. A primeira tarefa não é abrir o n8n e montar blocos. A primeira tarefa é entender o processo.

Conceitos principais

ConceitoExplicaçãoExemplo
AutomaçãoExecução automática de tarefa repetitivaEnviar confirmação
Automação inteligenteAutomação que usa IA em uma ou mais etapasClassificar mensagem livre
WorkflowFluxo automatizado de etapasFormulário → planilha → e-mail
PipelineSequência de processamento de dadosReceber → validar → padronizar → salvar
SkillCapacidade reutilizávelConsultar certificado

Diferença entre automação simples e automação inteligente

Automação simples:
Se o campo e-mail estiver preenchido, salvar na planilha.
Se o campo e-mail estiver vazio, registrar erro.

Automação inteligente:
Ler uma mensagem escrita livremente e classificar se ela é sobre inscrição, certificado, dúvida ou suporte técnico.

Exemplo aplicado ao UNESP.IA

Workflow:

Formulário → n8n → planilha → mensagem de confirmação → e-mail → log

Pipeline dentro do workflow:

Receber dados → validar campos obrigatórios → padronizar nome → normalizar módulo → classificar perfil → salvar dados
Uso possível de LLM: gerar uma mensagem personalizada de boas-vindas. Onde não usar LLM: verificar se o campo e-mail está vazio.

Roteiro de implementação no n8n

  1. Criar um workflow com Manual Trigger.
  2. Criar dados fictícios com Edit Fields.
  3. Inserir campos como nome, e-mail, módulo e perfil.
  4. Criar uma validação simples com IF.
  5. Registrar o resultado como válido ou pendente.
  6. Identificar quais partes representam processo, workflow, pipeline, skill e possível uso de LLM.

Produto da unidade

Mapa inicial de uma automação inteligente.

Evidência da competência

O participante demonstra que compreende a diferença entre processo, workflow, pipeline, skill e uso de IA.

Para refletir

A automação deve começar pela ferramenta ou pelo entendimento do processo?

12.2 Conceitos essenciais: workflow, pipeline, prompt, skill, tool, LLM, chatbot e agente

Carga horária

3 horas.

Objetivo da unidade

Diferenciar os conceitos centrais usados em automações inteligentes, evitando confusão entre prompt, skill, workflow, pipeline, tool, LLM, chatbot e agente.

Situação-problema

Ana deseja criar uma solução para responder participantes que perguntam sobre certificados. A solução envolve receber mensagem, identificar intenção, validar e-mail, consultar planilha, verificar frequência, verificar status, gerar resposta e registrar log. Essa solução não é apenas um prompt. Trata-se de uma skill, que pode usar um prompt em uma etapa específica.

Conceitos principais

ConceitoO que éExecuta ação?Pode usar IA?Exemplo
PromptInstrução enviada para uma LLMNãoSim“Classifique esta mensagem”
LLMModelo de linguagemNão diretamenteSimGPT, Gemini, Claude
ToolFerramenta acionávelSimPode ou nãoGmail, Sheets, API
SkillCapacidade reutilizávelSimPode ou nãoConsultar certificado
WorkflowFluxo completoSimPode ou nãoInscrição → planilha → e-mail
PipelineSequência de processamento de dadosSimPode ou nãoValidar → limpar → salvar
ChatbotInterface conversacionalPode acionar açõesPode ou nãoChat de atendimento
AgenteSistema com LLM, tools, skills e regrasSim, via tools/skillsSimAgente de atendimento

Diferenças principais

Prompt ≠ Skill
Prompt orienta a IA. Skill é uma capacidade completa e reutilizável.

Workflow ≠ Pipeline
Workflow representa o fluxo completo. Pipeline representa o tratamento dos dados dentro do fluxo.

Tool ≠ Skill
Tool é a ferramenta acionável. Skill é a capacidade funcional que usa uma ou várias tools.

Chatbot ≠ Agente
Chatbot é interface de conversa. Agente interpreta, escolhe tools ou skills e executa ações dentro de limites.

Exemplo aplicado ao UNESP.IA

ParteConceito
Texto do participanteEntrada
Classificar como certificadoPrompt + LLM
Consultar situação do certificadoSkill
Buscar dados na planilhaTool
Validar frequência e statusPipeline
Receber, classificar, consultar e responderWorkflow
Escolher a skill corretaAgente

Roteiro de implementação no n8n

  1. Criar um fluxo manual com uma mensagem de exemplo.
  2. Criar um node de IA para classificar a mensagem.
  3. Solicitar saída em JSON.
  4. Usar Switch para separar categorias.
  5. Registrar a categoria em planilha.
  6. Documentar prompt, LLM, tool, skill, pipeline e workflow.

Produto da unidade

Quadro comparativo e miniworkflow de classificação.

Para refletir

12.4 Primeiro workflow com IA

Carga horária

2 horas.

Objetivo da unidade

Criar o primeiro workflow com uso pontual de LLM para gerar uma resposta personalizada.

Workflow implementado

Manual Trigger → Edit Fields → LLM → Resultado

Entrada

{
  "nome": "Maria Oliveira",
  "perfil": "Comunidade externa",
  "modulo": "Fundamentos e Letramento em IA"
}

Prompt utilizado

Você é um assistente educacional do projeto UNESP.IA.

Gere uma mensagem curta, clara, acolhedora e institucional
para uma pessoa que acabou de se inscrever no curso.

Dados:
Nome: {{nome}}
Perfil: {{perfil}}
Módulo: {{modulo}}

A mensagem deve:
1. cumprimentar a pessoa pelo nome;
2. confirmar a inscrição;
3. mencionar o módulo escolhido;
4. usar linguagem simples;
5. não prometer certificado automaticamente;
6. finalizar com uma frase de boas-vindas.

Onde usar LLM

  • quando a mensagem precisa variar por perfil;
  • quando há diferentes públicos;
  • quando o texto precisa ser acolhedor;
  • quando há necessidade de adaptação de linguagem.

Onde não usar LLM

  • quando a mensagem pode ser fixa;
  • quando o volume é grande e o custo precisa ser reduzido;
  • quando há risco de prometer algo indevido.

Roteiro de implementação no n8n

  1. Criar Manual Trigger.
  2. Adicionar Edit Fields com nome, perfil e módulo.
  3. Adicionar node de LLM.
  4. Inserir o prompt.
  5. Executar com três perfis diferentes.
  6. Comparar resposta fixa versus resposta gerada por IA.
  7. Registrar no log se o fluxo usou LLM.

Produto da unidade

Workflow simples com geração de mensagem por LLM.

Exemplo de skill — consultar_certificado

12.7 LLMs, prompts e saídas estruturadas

Carga horária

3 horas.

Objetivo da unidade

Mostrar onde as LLMs devem ser usadas dentro de automações no n8n, onde devem ser evitadas e como criar prompts estruturados com saídas controladas.

Usar LLM quando

  • houver texto livre;
  • houver ambiguidade;
  • for necessário classificar intenção;
  • for necessário resumir conteúdo;
  • for necessário extrair campos;
  • for necessário gerar resposta personalizada;
  • for necessário transformar texto em JSON.

Evitar LLM quando

  • houver regra simples;
  • houver cálculo objetivo;
  • houver decisão institucional;
  • houver dado oficial a consultar;
  • houver informação sensível;
  • bastar um template fixo.

Prompt estruturado

Classifique a mensagem abaixo em uma das categorias:
inscricao, certificado, duvida_conteudo, suporte_tecnico, sugestao, estoque, pedido, manutencao, outro.

Mensagem:
{{mensagem}}

Responda somente em JSON:
{
  "categoria": "",
  "prioridade": "baixa|media|alta",
  "justificativa": "",
  "precisa_humano": true|false
}

Workflow

Mensagem recebida → LLM classifica → validar JSON → Switch por categoria → executar skill adequada → registrar log

Roteiro de implementação no n8n

  1. Criar entrada com mensagens variadas.
  2. Enviar a mensagem para a LLM.
  3. Solicitar saída em JSON.
  4. Validar se os campos esperados foram retornados.
  5. Usar Switch por categoria.
  6. Registrar resultado em planilha.
  7. Encaminhar para skills diferentes conforme a categoria.

Produto da unidade

Workflow com LLM, saída JSON validada e roteamento por categoria.

12.8 Aplicações no UNESP.IA

Aplicação 1 — Inscrição automática

Skills envolvidas: registrar_inscricao, classificar_perfil_participante, enviar_confirmacao, registrar_log_execucao.

Básico

Problema que resolve: Registrar automaticamente participantes inscritos em curso, módulo, oficina ou atividade, reduzindo retrabalho manual e erros de cadastro.

Entradas: nome, e-mail, cidade, perfil, módulo, data da inscrição, autorização de uso dos dados no contexto do curso.

Tools utilizadas: Webhook, Google Forms, Google Sheets, IF, Switch, Edit Fields e log em planilha.
